Introduction

This document outlines the API endpoints, their purposes, input parameters, and expected outputs. Use this documentation as a reference for integrating with the API.

Base URL

https://api.example.com/v1

Endpoints

1. GET /users

Description: Retrieve a list of users.

Parameters:

  • page (optional, int): Page number for pagination.
  • limit (optional, int): Number of users per page.

Response:


{
    "status": "success",
    "data": [
        { "id": 1, "name": "John Doe", "email": "john.doe@example.com" },
        { "id": 2, "name": "Jane Doe", "email": "jane.doe@example.com" }
    ]
}

2. POST /users

Description: Create a new user.

Request Body:


{
    "name": "John Doe",
    "email": "john.doe@example.com",
    "password": "securepassword123"
}

Response:


{
    "status": "success",
    "message": "User created successfully.",
    "data": { "id": 3, "name": "John Doe", "email": "john.doe@example.com" }
}

3. GET /users/{id}

Description: Retrieve details of a specific user.

Parameters:

  • id (required, int): The ID of the user.

Response:


{
    "status": "success",
    "data": { "id": 1, "name": "John Doe", "email": "john.doe@example.com" }
}

4. PUT /users/{id}

Description: Update a user's information.

Request Body:


{
    "name": "John Smith",
    "email": "john.smith@example.com"
}

Response:


{
    "status": "success",
    "message": "User updated successfully."
}

5. DELETE /users/{id}

Description: Delete a user.

Parameters:

  • id (required, int): The ID of the user.

Response:


{
    "status": "success",
    "message": "User deleted successfully."
}

Error Handling

All error responses will follow this structure:


{
    "status": "error",
    "message": "Error description",
    "code": 400
}

Authentication

The API uses token-based authentication. Include a header in your requests: 

Authorization: Bearer {token}

Rate Limiting

Each user is allowed 1000 requests per hour. Exceeding this limit will result in a 429 Too Many Requests error.
